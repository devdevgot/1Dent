START: Пациент пишет в чат
│
├── 1. Классификация запроса
│   │
│   ├── Определить услугу
│   │   ├── ❓ Неясно / консультация
│   │   ├── 🦷 Терапия
│   │   ├── 🪥 Чистка
│   │   ├── 🩺 Хирургия
│   │   ├── 🦷 Ортопедия
│   │   └── 😁 Ортодонтия
│   │
│   ├── Определить срочность
│   │   ├── 🚨 Срочно
│   │   ├── ⚡ Скоро
│   │   └── 🕐 Планово
│   │
│   ├── Определить уверенность
│   │   ├── Низкая
│   │   └── Высокая
│   │
│   └── Определить тип пациента
│       ├── Новый
│       ├── Повторный
│       └── VIP (высокий чек)
│
├── 2. Выбор маршрута
│   │
│   ├── ЕСЛИ уверенность = Низкая
│   │   └── → Терапевт (консультация)
│   │
│   └── ЕСЛИ уверенность = Высокая
│       └── → Прямая запись к специалисту
│
├── 3. Фильтр кандидатов
│   │
│   ├── По специализации
│   ├── По времени (± окно)
│   ├── По статусу (работает)
│   └── По минимальной загрузке (исключить перегруженных)
│
├── 4. Проверка срочности
│   │
│   ├── ЕСЛИ Срочно
│   │   └── → Выбрать ближайшего врача
│   │       └── (игнор score и квот)
│   │
│   └── ЕСЛИ НЕ срочно
│       └── → Перейти к скорингу
│
├── 5. Расчет факторов (advanced scoring)
│   │
│   ├── base_score (качество врача)
│   │   ├── выручка
│   │   ├── чек
│   │   ├── конверсия
│   │   ├── скорость
│   │   └── надежность
│   │
│   ├── real_value_factor
│   │   └── (пришел × оплатил × чек)
│   │
│   ├── quota_factor (динамический баланс)
│   │   └── target_share / current_share
│   │
│   ├── time_factor
│   │   └── близость слота
│   │
│   ├── load_factor
│   │   ├── текущая загрузка
│   │   └── прогноз загрузки (вперед на 1–2 часа)
│   │
│   ├── value_factor (деньги)
│   │   ├── дорогая услуга → буст топ врачам
│   │   └── простая → баланс
│   │
│   ├── patient_fit_factor
│   │   ├── новый → врач с высокой конверсией
│   │   ├── постоянный → тот же врач
│   │   └── VIP(Поиск по ключевым словам дорогостоющих услуг) → топ врач
│   │
│   ├── exploration_factor
│   │   ├── 60% → лучший врач
│   │   └── 40% → случайный из топ-N
│   │
│   └── final_score =
│       base × real_value × quota × time × load × value × fit × exploration
│
├── 6. Выбор врача
│   │
│   ├── max(final_score)
│   │
│   └── Ограничения:
│       ├── не более X подряд за день
│       ├── лимит % в час
│       └── минимальная гарантия загрузки (для слабых)
│
├── 7. Запись пациента
│   │
│   └── Создать appointment
│
├── 8. Прием пациента
│   │
│   ├── Врач принимает
│   │   └── → фиксируем:
│   │       ├── пришел / не пришел
│   │       ├── оплатил / нет
│   │       └── сумма
│   │
│   └── ИЛИ врач передает (handoff)
│       │
│       ├── система рекомендует врача (AI)
│       ├── врач выбирает
│       │
│       ├── создать handoff
│       └── создать новый appointment
│
├── 9. Учет передачи
│   │
│   ├── handoff_out → НЕ влияет на рейтинг
│   ├── handoff_in → влияет
│   │
│   └── логирование причин:
│       ├── не его специализация
│       ├── сложный случай
│       └── пожелание пациента
│
├── 10. Feedback loop (самое важное)
│   │
│   ├── обновить real_value врача
│   ├── обновить конверсию
│   ├── обновить чек
│   └── обновить модель
│
└── 11. Аналитика и оптимизация
    │
    ├── % передач по врачам
    ├── доход на врача
    ├── эффективность по услугам
    ├── узкие места
    ├── прогноз загрузки
    └── рекомендации:
        ├── кого бустить
        ├── кого разгрузить
        └── где теряются деньги
