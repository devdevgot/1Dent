Промпт

У нас есть готовая страница «Пользователи» (/users) в CRM для стоматологической
клиники. Нужно её переработать — не создавать с нуля, а обновить существующую.

ЗАДАЧА 3 — НОВЫЙ FLOW ДОБАВЛЕНИЯ СОТРУДНИКА (4 шага):

Шаг 1 — «Email»
- Одно поле: введите email сотрудника
- Крупный инпут по центру
- Подпись: «На этот адрес придёт приглашение войти в систему»
- Валидация формата email inline
- Кнопка «Далее»

Шаг 2 — «Информация о сотруднике»
- ФИО (обязательно)
- Телефон (необязательно)
- Визуальный выбор роли — большие карточки с иконкой и описанием:
  · Администратор — управляет клиникой и расписанием
  · Врач — ведёт приём пациентов
  · Бухгалтер — видит финансы
  · Склад — управляет материалами
- Если выбран «Врач» — появляется поле специализации и
  максимальное количество пациентов в день
- Дата приёма на работу
- Кнопки «Назад» и «Далее»

Шаг 3 — «Зарплата»
- Четыре варианта (визуальный выбор карточками):
  · Оклад — поле суммы в месяц
  · Процент — поле процента от выручки
  · Оклад + Процент — оба поля одновременно
  · Почасовая — поле суммы за час
- При выборе варианта — соответствующие поля появляются сразу под карточками
- Кнопки «Назад» и «Далее»

Шаг 4 — «Отправить приглашение»
- Саммари: имя, роль, email сотрудника
- Текст: «Сотрудник получит письмо с временным паролем и ссылкой для входа»
- Одна кнопка: «Добавить и отправить приглашение»
- После успеха — тост: «Сотрудник добавлен, приглашение отправлено на [email]»
- После ошибки — тост: «Не удалось добавить сотрудника. Попробуйте ещё раз»

Прогресс-бар вверху диалога (4 шага).
Закрыть можно в любой момент — если данные введены, показать подтверждение.

ЗАДАЧА 4 — BACKEND (новый endpoint):

POST /api/users/invite
Доступ: owner, admin
Логика:
1. Создать сотрудника с переданными данными
2. Сгенерировать временный пароль (8 символов, случайный)
3. Захешировать через bcrypt, сохранить в БД
4. Отправить письмо через существующий mail-сервис:
   — Название клиники
   — Имя сотрудника
   — Временный пароль
   — Ссылка на вход
   — Инструкция: сменить пароль после первого входа
5. Вернуть { success: true }

Обновить модель зарплаты — добавить новый тип:
- hourly: { type: 'hourly', hourlyRate: number }

Итоговые типы зарплаты: fixed / percent / fixed_percent / hourly

Ограничение: повторная отправка приглашения не чаще раза в 60 секунд на один email.
