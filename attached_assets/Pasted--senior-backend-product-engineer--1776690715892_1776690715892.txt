Ты — senior backend архитектор и product engineer. Твоя задача — спроектировать и описать реализацию системы сквозной аналитики для клиник, где основной канал коммуникации — WhatsApp.

Цель системы: создать систему, которая генерирует реферальные ссылки для разных источников (Instagram, блогеры, реклама), отслеживает каждый клик, автоматически редиректит пользователя в WhatsApp клиники, передаёт в сообщение уникальный click_id, связывает входящие сообщения с источником трафика и показывает аналитику по каналам (клики, диалоги, конверсии).

Общая логика работы: пользователь кликает по ссылке вида https://yourdomain.com/r/{ref_id}. Backend находит кампанию по ref_id, создаёт уникальный click_id (UUID), сохраняет клик в базе данных (campaign_id, click_id, ip, user_agent, utm-параметры). После сохранения формируется ссылка WhatsApp вида https://wa.me/{phone}?text=…, куда добавляется click_id и источник. Далее происходит редирект: если номер есть — в WhatsApp, если нет — на fallback (лендинг или форма). Когда пользователь пишет в WhatsApp, система через webhook от Green API принимает сообщение, извлекает click_id из текста и связывает чат с кликом.

Критически важно: нельзя проверять подключение WhatsApp перед редиректом, нельзя зависеть от статуса Green API, редирект должен происходить всегда, даже если WhatsApp не работает — клик должен быть сохранён.

Требуется описать архитектуру системы (backend на Node.js или NestJS, база PostgreSQL, обработка webhook, масштабируемость), структуру базы данных (таблицы campaigns, clicks, messages с полями id, ref_id, click_id, campaign_id, whatsapp_number, fallback_url, ip, user_agent, utm и т.д.), реализацию endpoint GET /r/:ref (поиск кампании, создание click_id, сохранение клика, формирование ссылки, редирект), формирование правильной WhatsApp ссылки (без плюсов и пробелов, с encode текста), обработку webhook (приём сообщения, regex для извлечения click_id, сохранение и связывание), fallback-логику (лендинг, форма, альтернативный мессенджер), защиту от ошибок (невалидный номер, пустой ref_id, дубли кликов, повторные переходы).
