Задача: подготовить веб-приложение (SPA / React / Vue / Next.js) для корректного снятия полного скриншота страницы (full-page screenshot), чтобы весь контент отображался как единая длинная страница без внутренних скроллов и обрезаний.

Требования:

1. Убрать все ограничения, связанные с прокруткой:
- У всех контейнеров (включая html, body и корневые div) убрать:
  overflow: hidden;
  overflow: auto;
  overflow: scroll;
- Заменить на:
  overflow: visible !important;

2. Убрать фиксированную высоту:
- Найти все элементы с:
  height: 100vh;
  height: 100%;
  max-height: 100vh;
- Заменить на:
  height: auto !important;
  max-height: none !important;

3. Убрать внутренние скроллы:
- Все scroll-контейнеры должны “распаковаться” в общий поток страницы
- Контент должен растягивать страницу вниз, а не скроллиться внутри блока

4. Отключить позиционирование, мешающее скриншоту:
- Для элементов с:
  position: fixed;
  position: sticky;
- Заменить на:
  position: static !important;

5. Отключить виртуализацию списков (если используется):
- Если используется react-window / react-virtualized:
  - отрендерить ВСЕ элементы списка сразу
  - отключить lazy rendering

6. Отключить lazy loading:
- Убрать loading="lazy" у изображений
- Принудительно загрузить весь контент

7. Добавить print-режим (обязательно):
Добавить CSS:

@media print {
  html, body, #root {
    height: auto !important;
    overflow: visible !important;
  }

  * {
    overflow: visible !important;
    height: auto !important;
    max-height: none !important;
  }

  .no-print {
    display: none !important;
  }
}

8. Убедиться:
- Вся страница рендерится одной длинной колонкой
- Нет обрезанных блоков
- Нет внутренних скроллов
- Весь контент доступен при обычном scroll страницы

Цель:
После изменений страница должна корректно сниматься инструментами:
- Chrome full page screenshot
- Puppeteer / Playwright
- плагины для скриншотов
- экспорт в PDF (print)

Результат:
Один длинный скрин всей страницы, пригодный для импорта в Figma.
